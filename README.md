# 男朋友养成计划 V3.1 Cloud ❤️

这是一个专为情侣设计的互动网页应用，包含甜蜜积分、愿望清单、恋爱商店、**温馨相册**、**姨妈助手**等功能，并支持云端数据同步。

## 📁 项目结构

本项目已拆分为多个HTML页面，便于在GitHub Pages上部署和访问：

- **`index.html`** - 主页面（积分系统、每日签到、加减分功能）
- **`wishlist.html`** - 愿望清单页面（记录和完成恋爱愿望）
- **`album.html`** - 温馨相册页面（拍立得风格照片管理）
- **`shop.html`** - 恋爱商店页面（兑换中心和背包系统）
- **`settings.html`** - 设置页面（生理期记录、云同步配置、历史记录）

### 公共资源文件

- **`common.css`** - 所有页面共享的样式文件
- **`common.js`** - 所有页面共享的JavaScript逻辑和数据管理

### 页面导航

所有页面顶部都有统一的导航栏，可以快速切换不同功能模块。每个页面都独立运行，但共享相同的数据存储（通过LeanCloud云端同步或本地存储）。

## 📱 如何安装到手机？

### 方法一：添加到主屏幕（最推荐，永久免费）🌟
这种方法不需要复杂的打包，不需要证书，体验和原生 App 几乎一样（全屏运行，无地址栏）。

1.  将所有HTML文件（`index.html`, `wishlist.html`, `album.html`, `shop.html`, `settings.html`）和公共资源文件（`common.css`, `common.js`）上传到任意免费静态网站托管（如 GitHub Pages, Vercel, 或自己的服务器）。
2.  在苹果手机 Safari 浏览器中打开该链接。
3.  点击底部中间的 **分享按钮**（带有向上箭头的方框）。
4.  下滑找到并点击 **“添加到主屏幕”**。
5.  点击右上角的 **“添加”**。
6.  桌面就会出现一个爱心图标的 App，点击即可全屏运行！

### 方法二：打包成安装包 (IPA) 🛠️
如果你坚持想要一个 `.ipa` 文件通过爱思助手安装，由于苹果系统的封闭性，步骤会比较复杂，且通常需要**付费购买证书**。

**准备工作：**
*   电脑安装 **HBuilderX** (uni-app 官方开发工具)。
*   准备苹果开发者证书（Certificate）和描述文件（Provisioning Profile）。（通常需要苹果开发者账号 $99/年，或者在淘宝/第三方平台购买个人签名证书）。

**打包步骤：**
1.  打开 HBuilderX，选择“文件” -> “新建” -> “项目”。
2.  选择 “5+App” -> “默认模板”，输入项目名称。
3.  将本项目中的 `index.html` 内容复制并覆盖新建项目中的 `index.html`。
4.  打开 `manifest.json`，配置 App 图标、启动图等信息。
5.  点击顶部菜单 “发行” -> “原生App-云打包”。
6.  选择 “iOS (ipa包)”，填入你的证书信息（证书私钥密码、profile文件等）。
7.  点击“打包”，等待完成后下载 IPA 文件。
8.  使用爱思助手连接手机，将 IPA 文件导入安装。

## 🚀 GitHub Pages 部署

本项目已优化为多页面结构，非常适合部署到GitHub Pages：

1. 将所有文件推送到GitHub仓库
2. 在仓库设置中启用GitHub Pages
3. 选择主分支（通常是`main`或`master`）作为源
4. 访问 `https://你的用户名.github.io/仓库名/` 即可使用

### 访问方式

- 主页：`https://你的用户名.github.io/仓库名/index.html` 或 `https://你的用户名.github.io/仓库名/`
- 愿望清单：`https://你的用户名.github.io/仓库名/wishlist.html`
- 相册：`https://你的用户名.github.io/仓库名/album.html`
- 商店：`https://你的用户名.github.io/仓库名/shop.html`
- 设置：`https://你的用户名.github.io/仓库名/settings.html`

## ⚠️ 注意事项
*   本项目使用了 LeanCloud 进行数据存储，请确保在设置页面中正确配置了你的 AppID 和 AppKey。
*   iOS 系统对隐私权限要求严格，如果遇到无法通过爱思助手安装的情况，通常是证书失效或未信任导致。
*   **相册说明**：为了保证同步速度，本地上传的图片会自动进行压缩，且建议相册存储照片数量控制在 20 张以内。
*   **多页面说明**：所有页面共享相同的数据存储，通过`common.js`统一管理。每个页面都可以独立访问，但数据会实时同步。
